需要关注的问题

P0 样式系统双层统治 — ConfigProvider vs CSS 选择器职责冲突 main.tsx + index.css
ConfigProvider 覆盖 40+ 组件 token，index.css 又通过 CSS 选择器覆盖 20+ 组件样式。两套系统在争夺真相。.ant-btn 的 font-weight: 500 在 CSS 里，主题配置也有 Button.lineHeight。维护者无法判断哪个"赢了"。
建议: 统一到 ConfigProvider token 层，或反之一致使用 CSS 选择器，不要双线作战。
P0 登录页装饰过度 — 与"冷静克制"品牌人格冲突 login/index.tsx
6 个动画元素 (blob ×3 + 网格 + 光圈 ×3 + bounce) — 更像 SaaS landing page 而非管理工具。品牌规则禁止蓝/紫色系，但用了 bg-blue-300 和 bg-indigo-300。登录按钮渐变 from-teal-500 to-emerald-500 在工具类产品中多余。
建议: 移除 blob 渐变动画，登录按钮改用纯色 Teal。让登录页像"操作台入口"。
P1 登录表单混用原生 HTML 输入 vs Ant Design Form login/index.tsx:266
登录表单用原生 <input> + Tailwind 手写，同一页面的"账号申请" Modal 用 Ant Design Form。同一用户旅程中两套表单词汇。
建议: 统一用 Ant Design Form，或至少在验证、高度、焦点态上保持一致。
P1 按钮样式碎片化 — 绕过 ConfigProvider 主题 llm-management/index.tsx:464
多处手动写 bg-brand-600 border-brand-600 hover:bg-brand-700... 完全绕过主题配置。部分按钮用 type="primary"，部分用 Tailwind 硬写。
建议: 统一用 Button type="primary"，主题配置管一切。
P1 登录页内联 <style> 标签 — blob / spin keyframes login/index.tsx:160
keyframe 动画写在组件 JSX 的 <style> 中，每次 re-render 重新注入 CSS，破坏样式表职责边界。
建议: 迁移到 index.css 或 login.module.css。
P1 内容区域使用百分比宽度 — w-[96%] px-[2%] dashboard-layout.tsx:684
百分比宽度组合不响应固定断点，小屏幕边距过小，大屏幕边距过大。
建议: 改用 max-w-7xl mx-auto 或 px-4 sm:px-6 md:px-8。
P2 Modal 宽度不统一 — px / vw / default 混用 多页面
640 / "58vw" / 520 / 默认 — 四种方式，无标准。
建议: 统一定义 2–3 档标准宽度 (520 小表单 / 640 标准 / 720 宽表单)。
P2 登录卡片 rounded-3xl (24px) 超出品牌圆角上限 login/index.tsx:201
品牌规范上限 12–14px，登录卡片用了 24px。登录页不应打破全局约定。
建议: 改用 rounded-2xl (16px) 或 rounded-xl (12px)。
P2 侧栏"服务运行中" — 虚假状态指示器 dashboard-layout.tsx:350
静态显示绿色 + ping 动画，无真实健康检查支撑。后端宕机时仍然显示正常。
建议: 接入真实健康检查接口，或改为"已连接"语义。
P3 自定义滚动条三类高度重复 index.css:285-348
task-command-modal / scrolling-text-form-modal / custom-scrollbar 高度重复。
建议: 合并为统一 .scrollbar-teal 工具类。

总结与优先级建议

整体评价：这是一个质量上乘的管理后台，品牌色 (Teal) 执行一致，响应式布局合理，表格/表单等组件使用克制。

核心问题有两个：

  1. 样式系统冗余（ConfigProvider + CSS 选择器双线作战）— 影响所有页面，增加维护成本
  2. 页面间组件词汇不一致（登录 vs 管理页的表单、按钮、圆角）— 导致视觉碎片化

需要优先处理的是 P0 样式系统统一 和 P0 登录页装饰过度 — 前者影响全局，后者违背了"冷静克制"的产品品牌人格。
