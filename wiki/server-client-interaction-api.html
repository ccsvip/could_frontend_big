服务端与客户端交互接口文档

本文档面向前端、安卓端和后端维护人员，集中说明 HTTP API、统一 WebSocket 命令协议、设备在线状态、ASR 流式识别和 TTS 流式播放的对接方式。 示例只展示协议结构，所有 token、密码、密钥均使用占位符。

HTTP 基地址 /api/v1
统一 WebSocket /ws/realtime/
后台客户端 Bearer JWT
设备运行时 deviceCode

交互边界

当前系统有三类一方客户端：后台 Web 管理端、安卓设备端、AI 语音测试或应用端。HTTP 负责配置、资源、管理和兼容调用；实时事件、在线状态、ASR 音频流、TTS 音频流统一走一个 WebSocket 入口。

后台 Web 管理端

登录后携带 Bearer JWT，管理租户、设备、资源、知识库、AI 模型和命令配置。超管在公司视图下访问业务接口时可追加 tenant 参数。

Authorization: Bearer <access_token>

安卓设备端

设备端不登录后台，不持有后台 JWT。设备身份使用 deviceCode，激活和配置走 HTTP，在线状态走统一 WebSocket 的 device.status 命令。

{ "deviceCode": "ANDROID-BOARD-001" }

AI 语音端

ASR 和 TTS 的后台配置走 HTTP；实时音频交互走统一 WebSocket，并通过 asr.session.* / tts.session.* 命令创建或结束会话。

{ "type": "asr.session.start", "id": "asr-1" }

服务端

服务端负责权限校验、租户隔离、设备生命周期、资源聚合、语音上游连接和统一错误返回。新实时能力应优先扩展命令协议。

/api/v1 + /ws/realtime/
在线 / 离线状态以 WebSocket 连接生命周期为准。POST /device-runtime/heartbeat/ 只保留兼容字段更新，不再决定设备 online / offline。

端到端时序

下面是最常见的设备运行链路：后台先配置并授权，设备再激活、拉配置、建立实时连接，后台通过同一个 WebSocket 订阅设备变化。

客户端
  1. 后台登录，获取 access token。
  2. 后台创建应用、资源、命令组并授权设备。
  3. 安卓端用 deviceCode 激活或上报设备。
  4. 安卓端拉取运行配置。
  5. 安卓端连接 /ws/realtime/ 并发送 device.status.start。
  6. 后台端连接 /ws/realtime/ 并发送 devices.events.subscribe。
→
服务端
  1. 签发 JWT 并按角色返回用户上下文。
  2. 保存租户、资源、设备、应用关系。
  3. 登记设备档案，未知设备进入待授权列表。
  4. 按 deviceCode 聚合应用和资源包。
  5. 连接建立时置 online，断开时置 offline。
  6. 把设备绑定、撤销、上线、离线事件推送给订阅端。

认证接口

认证接口挂载在 /api/v1/auth/ 下，后台 Web 管理端使用。示例中的用户名、密码和 token 均为占位符。

登录请求 POST 
{
  "url": "/api/v1/auth/login/",
  "headers": {
    "Content-Type": "application/json"
  },
  "body": {
    "username": "<后台用户名>",
    "password": "<后台密码>"
  }
}
登录响应 200 
{
  "access": "<access_token>",
  "refresh": "<refresh_token>",
  "user": {
    "id": 1,
    "username": "<username>",
    "role": {
      "code": "admin",
      "name": "管理员"
    },
    "tenant": null,
    "must_change_password": false
  }
}
方法 路径 客户端用途 服务端返回
POST /auth/login/ 后台账号登录。 access、refresh、用户角色、租户上下文。
POST /auth/refresh/ 使用 refresh token 换取新的 access token。 新的 access token。
GET /auth/me/ 读取当前登录用户信息。 用户、角色、租户、权限相关信息。
POST /auth/change-password/ 修改当前账号密码。 修改结果。
GET / POST /auth/account-applications/ 读取或创建账号应用授权关系。 账号应用数据。

AI 与语音接口

后台配置和测试接口仍然是 HTTP；真正的 ASR / TTS 流式媒体交互统一走 WebSocket 命令协议。

能力 HTTP 路径 客户端用途 备注
LLM 选项 /ai-models/llm/options/ 公司端读取可用模型。 受租户授权影响。
默认 LLM /ai-models/llm/default-model/ 读取或保存公司默认模型。 后台 Web 使用。
TTS 选项 /ai-models/tts/options/ 读取可用供应商和音色。 运行时 WebSocket 可传 voiceId / providerCode。
TTS 运行时 /ai-models/tts/runtime/ HTTP 形式的 TTS 调用。 流式播放优先使用 tts.session.start。
ASR 状态 /ai-models/asr/status/ 后台测试前读取 ASR 是否可用。 返回配置状态。
ASR 设备状态 /ai-models/asr/device-status/ 设备端按 deviceCode 验证 ASR 租户上下文。 运行时再连接 /ws/realtime/。

统一 WebSocket 协议

第一方实时通信只维护一个入口：/ws/realtime/。不同能力通过 JSON 命令的 type 字段区分，二进制帧只在 ASR 会话启动后用于上传音频。

客户端命令包 WS 
{
  "type": "命令名称",
  "id": "客户端生成的相关 ID",
  "payload": {
    "token": "<access_token 或实时票据>",
    "tenantId": 2,
    "deviceCode": "ANDROID-BOARD-001"
  }
}
服务端事件包 WS 
{
  "type": "结果或事件名称",
  "id": "回传请求中的相关 ID",
  "payload": {},
  "error": {
    "code": "invalid_command",
    "message": "Realtime command type is required"
  }
}
新实时能力不要再新增功能专用 WebSocket URL。只有存在明确传输层原因时，例如不兼容的第三方协议、独立二进制流隔离、特殊安全边界，才考虑新增入口并记录原因。

命令清单

以下命令来自当前 /ws/realtime/ 服务端实现。客户端必须带稳定 type 和 id，服务端用 id 做响应关联。

命令 type payload 服务端成功事件 说明
ping {} pong 连接保活或探测。
devices.events.subscribe { token, tenantId? } devices.events.subscribed 后台设备页订阅设备事件。超管公司视图传 tenantId。
devices.events.unsubscribe {} devices.events.unsubscribed 取消设备事件订阅。
device.status.start { deviceCode } device.status.started 设备运行时上线，服务端发布 device.status 事件。
device.status.ping {} device.status.pong 设备在线连接保活并更新 lastHeartbeat。
asr.session.start { token, tenantId?, deviceCode? } asr.ready 启动 ASR 会话，之后客户端发送二进制音频帧。
asr.session.finish {} asr.done 通知 ASR 上游结束输入，等待最终转写。
asr.session.cancel {} asr.cancelled 取消 ASR 会话。
tts.session.start { token, tenantId?, text, voiceId?, providerCode? } tts.ready、二进制音频帧、tts.done 启动 TTS 合成并向客户端流式下发音频。
tts.session.cancel {} tts.cancelled 取消 TTS 合成会话。

  "type": "device.status.start",

    "deviceCode": "ANDROID-BOARD-001"
  }
}

ASR / TTS 流式交互

ASR 是客户端上传音频、服务端返回转写文本；TTS 是客户端发送文本、服务端返回音频二进制帧。两者都使用 /ws/realtime/。

ASR 客户端流程 WS 
{
  "connect": "ws://<host>/ws/realtime/",
  "start": {
    "type": "asr.session.start",
    "id": "asr-1",
    "payload": {
      "token": "<access_token 或实时票据>",
      "tenantId": 2,
      "deviceCode": "ANDROID-BOARD-001"
    }
  },
  "afterReady": [
    "发送音频二进制帧",
    "结束时发送 asr.session.finish",
    "取消时发送 asr.session.cancel"
  ]
}
ASR 服务端事件 WS 
{
  "ready": {
    "type": "asr.ready",
    "id": "asr-1"
  },
  "transcript": {
    "type": "asr.transcript",
    "id": "asr-1",
    "payload": {
      "text": "识别出的文本",
      "isFinal": false
    }
  },
  "done": {
    "type": "asr.done",
    "id": "asr-1"
  },
  "error": {
    "type": "asr.error",
    "id": "asr-1",
    "message": "ASR session is not authorized"
  }
}
TTS 客户端流程 WS 
{
  "connect": "ws://<host>/ws/realtime/",
  "start": {
    "type": "tts.session.start",
    "id": "tts-1",
    "payload": {
      "token": "<access_token 或实时票据>",
      "tenantId": 2,
      "text": "欢迎使用数字人终端",
      "voiceId": 12,
      "providerCode": "dashscope"
    }
  },
  "cancel": {
    "type": "tts.session.cancel",
    "id": "tts-cancel-1"
  }
}
TTS 服务端事件 WS 
{
  "ready": {
    "type": "tts.ready",
    "id": "tts-1",
    "sampleRate": 24000,
    "voice": "voice-code"
  },
  "binaryFrames": "服务端连续发送音频二进制帧",
  "done": {
    "type": "tts.done",
    "id": "tts-1"
  },
  "error": {
    "type": "tts.error",
    "id": "tts-1",
    "message": "TTS 服务未配置或未启用"
  }
}

错误与调试

HTTP 错误由 DRF 和业务视图返回；WebSocket 命令错误统一使用 error 事件，ASR / TTS 媒体错误使用对应领域事件。

WebSocket 通用错误 WS 
{
  "type": "error",
  "id": "client-command-id",
  "error": {
    "code": "unknown_command",
    "message": "Unsupported realtime command: demo.command"
  }
}
常见错误码 REF 
{
  "invalid_json": "命令不是合法 JSON",
  "invalid_command": "缺少 type 或 type 为空",
  "unknown_command": "服务端不支持该命令",
  "unauthorized": "token 或租户上下文无效",
  "invalid_device": "缺少 deviceCode",
  "device_not_found": "设备不存在、不可用或过期",
  "media_session_not_started": "未启动 ASR 会话却发送二进制帧"
}
现象 优先检查 处理建议
后台设备页没有实时刷新 是否连接 /ws/realtime/，是否发送 devices.events.subscribe。 确认 token 有效；超管公司视图确认 tenantId 已传。
设备一直 offline 安卓端是否发送 device.status.start，设备是否已启用。 不要依赖 HTTP heartbeat；检查 WebSocket 是否被代理转发。
ASR 没有转写 asr.session.start 是否收到 asr.ready，之后是否发送二进制音频帧。 先调用 ASR 状态接口确认服务配置，再检查 token、tenantId、deviceCode。
TTS 没有声音 tts.session.start 是否收到 tts.ready，浏览器是否播放二进制帧。 检查 voiceId、providerCode、TTS 配置和浏览器音频解码逻辑。

维护规则

这部分是给后续开发看的约束，避免接口越写越散。

实时能力先扩展命令

新增一方实时能力时，优先在 /ws/realtime/ 下新增 type 命令和明确 payload。不要用 URL 路径暗示业务行为。

请求必须可关联

WebSocket 命令必须带 id，服务端响应和错误要回传同一个 id，方便客户端并发处理。

租户上下文要显式

后台 JWT、tenantId、deviceCode 的使用边界要清晰。设备端不应持有后台账号密码或长期后台 JWT。

文档跟着协议走

新增或修改 HTTP 路径、WebSocket 命令、错误码时，同步更新本文档和 websocket-unification-progress.html。
