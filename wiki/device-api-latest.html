设备接口最新请求与响应

更新时间：2026-06-04；接口基地址：宿主 http://localhost:8880/api/v1，容器内 http://backend:8000/api/v1。

后台管理端需要 Bearer JWT；安卓运行时端不登录、不携带后台 JWT，只提交 deviceCode。

设备身份deviceCode
在线状态来源运行时 WebSocket
后台实时刷新管理端 WebSocket
HTTP 心跳用途兼容字段更新

关键规则

在线 / 离线状态不能再由 HTTP heartbeat 判定。安卓端应优先连接统一实时通信入口 `/ws/realtime/` 并发送 `device.status.start`；连接成功后才置为 online，连接断开才置为 offline。

{
  "rules": [
    "安卓端只传 deviceCode，不传 authCode、tenantId、后台 JWT、设备 token 或设备名称",
    "POST /device-auth/activate/ 只负责登记设备档案与更新版本、系统、主板等运行参数，不决定在线状态",
    "未知 deviceCode 首次上报时设备名称默认为“待修改”，由超管在设备授权中心双击名称后保存修改",
    "GET /device-runtime/config/ 只在设备已绑定可用应用后返回资源包",
    "POST /device-runtime/heartbeat/ 只兼容旧端更新 lastHeartbeat / softwareVersion / systemVersion",
    "后台设备管理页通过 /ws/realtime/ 发送 devices.events.subscribe 接收设备绑定、撤销、上线、离线事件后自动刷新"
  ]
}

POST安卓端激活 / 上报设备

请求路径：/api/v1/device-auth/activate/。未知 deviceCode 会创建待绑定设备，设备名称默认“待修改”；已存在设备只更新版本、系统、主板、deviceInfo 等运行参数，不采信安卓端设备名称。

{
  "method": "POST",
  "url": "http://localhost:8880/api/v1/device-auth/activate/",
  "headers": {
    "Content-Type": "application/json"
  },
  "body": {
    "deviceCode": "ANDROID-NEW-10004",
    "softwareVersion": "1.0.0",
    "systemVersion": "Android 14",
    "mainboardInfo": "rk3588",
    "deviceInfo": {
      "screen": "1080x1920",
      "network": "wifi"
    }
  }
}

WS安卓端运行时在线状态

连接地址：/ws/realtime/。连接后发送 device.status.start，这是当前决定 online/offline 的统一实时通信命令；旧设备状态专用 WebSocket 已退役，不要作为新入口使用。

{
  "url": "ws://localhost:8880/ws/realtime/",
  "start_command": {
    "type": "device.status.start",
    "id": "device-status-ANDROID-DEMO-10001",
    "payload": {
      "deviceCode": "ANDROID-DEMO-10001"
    }
  },
  "lifecycle": [
    "连接成功：后端校验设备存在、启用、未过期，然后写入 status=online 和 lastHeartbeat",
    "客户端可每 30 秒发送 device.status.ping，后端返回 device.status.pong 并更新 lastHeartbeat",
    "连接断开：后端写入 status=offline",
    "每次上线 / 离线都会发布后台设备事件，管理端设备列表无需刷新页面"
  ],
  "ping_command": {
    "type": "device.status.ping",
    "id": "device-ping-1"
  },
  "pong_event": {
    "type": "device.status.pong",
    "id": "device-ping-1",
    "payload": {
      "deviceCode": "ANDROID-DEMO-10001"
    }
  }
}

POST安卓端兼容心跳

请求路径：/api/v1/device-runtime/heartbeat/。保留给旧端更新 lastHeartbeat、softwareVersion、systemVersion；不再改变 online/offline。

{
  "method": "POST",
  "url": "http://localhost:8880/api/v1/device-runtime/heartbeat/",
  "headers": {
    "Content-Type": "application/json"
  },
  "body": {
    "deviceCode": "ANDROID-DEMO-10001",
    "softwareVersion": "1.0.1",
    "systemVersion": "Android 14",
    "deviceInfo": {}
  }
}

安卓端兼容心跳响应

{
  "status_code": 200,
  "body": {
    "status": "success",
    "message": "心跳成功"
  },
  "side_effect": "只更新 lastHeartbeat / 版本字段，不改变 status"
}

WS后台设备事件订阅

连接地址：/ws/realtime/。连接后发送 devices.events.subscribe；公司账号只会收到本公司事件，超管在公司视图必须传 tenantId。旧设备事件专用 WebSocket 已退役，不要作为新入口使用。

{
  "url": "ws://localhost:8880/ws/realtime/",
  "company_admin_command": {
    "type": "devices.events.subscribe",
    "id": "devices-subscribe-1",
    "payload": {
      "token": "***masked***"
    }
  },
  "super_admin_scoped_command": {
    "type": "devices.events.subscribe",
    "id": "devices-subscribe-tenant-2",
    "payload": {
      "token": "***masked***",
      "tenantId": 2
    }
  },
  "events": [
    "device.status：设备运行时 WebSocket 上线 / 离线",
    "device.authorization：超管绑定、再次授权、撤销授权"
  ],
  "frontend_behavior": "设备管理页收到 device.* 事件后自动刷新当前筛选条件下的列表和统计"
}

后台设备事件示例

{
  "status_event": {
    "type": "device.status",
    "tenantId": 2,
    "applicationId": 3,
    "deviceCode": "ANDROID-DEMO-10001",
    "status": "online",
    "isEnabled": true
  },

PATCH超管修改设备名称

请求路径：/api/v1/device-authorization-requests/{deviceCode}/name/。设备授权中心可双击“设备名称”进入编辑态，点击保存后更新后台设备名称；安卓端不传递该字段。

{
  "method": "PATCH",
  "url": "http://localhost:8880/api/v1/device-authorization-requests/ANDROID-NEW-10004/name/",
  "headers": {
    "Authorization": "Bearer ***masked***",
    "Content-Type": "application/json"
  },
  "body": {
    "name": "展厅数字人终端"
  },
  "response": {
    "deviceCode": "ANDROID-NEW-10004",
    "name": "展厅数字人终端",
    "bindingStatus": "pending"
  }
}

POST超管绑定待授权设备

请求路径：/api/v1/device-authorization-requests/{deviceCode}/bind/。绑定成功后会推送 device.authorization 事件，公司设备管理页自动出现该设备。

{
  "method": "POST",
  "url": "http://localhost:8880/api/v1/device-authorization-requests/ANDROID-NEW-10004/bind/",
  "headers": {
    "Authorization": "Bearer ***masked***",
    "Content-Type": "application/json"
  },
  "body": {
    "tenantId": 2,
    "applicationId": 3,
    "groupId": 4,
    "authorizationType": "permanent",
    "expiresAt": null,
    "isEnabled": true
  }
}

POST超管撤销授权

请求路径：/api/v1/device-authorization-requests/{deviceCode}/revoke/。撤销会写入 isEnabled=false、status=offline，并推送事件。

{
  "method": "POST",
  "url": "http://localhost:8880/api/v1/device-authorization-requests/ANDROID-NEW-10004/revoke/",
  "headers": {
    "Authorization": "Bearer ***masked***"
  },
  "response": {
    "deviceCode": "ANDROID-NEW-10004",
    "status": "offline",
    "isEnabled": false,
    "bindingStatus": "bound"
  }
}
