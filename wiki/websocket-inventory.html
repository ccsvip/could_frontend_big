WebSocket 服务端与客户端调用盘点

基于当前仓库源码整理。第一方实时通信统一入口为 /ws/realtime/，服务端通过消息 type 分发设备事件、设备在线状态、ASR、TTS、LLM 等能力。

统一入口 客户端命令 服务端回调 上游 WebSocket 生产客户端 调试与模拟器 数据结构 生命周期注意

1. 统一入口与路由

当前服务端没有 Django Channels consumer 分散入口，ASGI 层只把 /ws/realtime/ 转给 config.realtime.realtime_websocket_application。功能专用路径如 /ws/asr/test/、/ws/tts/test/ 不应新增或恢复。
层级 位置 职责 关键行为 来源文件
ASGI 路由 /ws/realtime/ 识别 WebSocket 请求并进入统一实时应用。 当 scope.type == websocket 且 path 匹配时调用 realtime_websocket_application，否则走普通 Django ASGI app。 backend/config/asgi.py
统一实时应用 realtime_websocket_application 接受连接、接收文本命令/二进制帧、转发设备事件队列、连接关闭清理。 连接后立即 accept；每条文本消息必须是 JSON object 且有字符串 type；二进制帧目前只作为 ASR PCM 音频上行。 backend/config/realtime.py
连接状态 RealtimeConnection 记录订阅、设备在线状态、ASR/TTS/LLM 当前 session 与任务。 断开时关闭 ASR/TTS/LLM、取消设备事件订阅，并把已启动在线状态的设备置为 offline 后发布 device.status。 backend/config/realtime.py
前端 URL 构造 buildRealtimeWebSocketUrl() 把 API 基地址转为 ws:/wss: 并固定 path。 清空 query，统一连接 /ws/realtime/。 web/src/api/realtime.ts

2. 客户端发往服务端的命令

命令 type 方向 payload / 参数 鉴权与租户 服务端处理与回调 用途 来源文件
ping 客户端 -> 服务端 无 payload；可带 id。 无。 返回 pong，结构为 {type,id,payload:{}}。 通用连接探活。 backend/config/realtime.py
devices.events.subscribe 后台管理端 -> 服务端 {token, tenantId?}，兼容 tenant。 JWT 用户；非超管需要 devices.view；普通公司用户固定为自身 tenant；超管可传 tenantId 收窄。 成功返回 devices.events.subscribed；后续设备事件以 devices.event 包装推送；失败返回通用 error/unauthorized。 后台设备管理页订阅设备上下线、授权变更等实时事件。 backend/config/realtime.py
backend/apps/devices/realtime.py
devices.events.unsubscribe 后台管理端 -> 服务端 无 payload；可带 id。 沿用当前连接订阅状态。 移除队列订阅，返回 devices.events.unsubscribed。 页面卸载或主动断开前取消设备事件订阅。 backend/config/realtime.py
web/src/api/realtime.ts
device.status.start 设备端 / 模拟器 -> 服务端 {deviceCode}，兼容 device_code。 通过设备码解析运行时设备，不需要后台 JWT。 设备置为 online，写心跳认证日志，发布 device.status online，并返回 device.status.started；缺设备码返回 invalid_device，设备不可用返回 device_not_found。 把设备在线状态绑定到 WebSocket 连接生命周期。 backend/config/realtime.py
backend/apps/devices/websocket.py
device.status.ping 设备端 / 模拟器 -> 服务端 无 payload；可带 id。 要求当前连接已执行 device.status.start。 更新 last_heartbeat，返回 device.status.pong；未 start 返回 device_status_not_started。 保持在线连接期间的最新心跳时间。 backend/config/realtime.py
backend/apps/devices/websocket.py
asr.session.start 后台页面 / 设备端 -> 服务端 后台：{token, tenantId?, requestId?, traceId?}。设备：{deviceCode, requestId?, traceId?}。兼容 snake_case。 后台 JWT 用户需要 ai_models.asr.view；设备通过 deviceCode 解析公司上下文。 解析 tenant 与 ASR 配置，连接上游 ASR WebSocket，发送 session.update，成功回 asr.ready；上游转写回 asr.transcript；上游结束回 asr.done；失败回 asr.error。 开始实时语音识别；后续客户端直接发送 16k mono PCM 二进制帧。 backend/config/realtime.py
backend/apps/ai_models/realtime_asr.py
asr.session.finish 后台页面 / 设备端 -> 服务端 无 payload；通常复用 ASR session id。 要求当前连接存在 ASR session。 向上游发送 session.finish；未开始则回 asr.error。 停止录音并提交本轮识别。 backend/config/realtime.py
web/src/api/realtime.ts
asr.session.cancel 后台页面 / 设备端 -> 服务端 无 payload；可带 cancel 命令 id。 要求当前连接存在 ASR session。 关闭上游连接与任务，返回 asr.cancelled，payload 带上一轮 sessionId；未开始则回 asr.error。 放弃当前识别，不提交后续音频。 backend/config/realtime.py
web/src/api/realtime.ts
tts.session.start 后台页面 / 设备端 -> 服务端 后台：{token, tenantId?, text, voiceId?, providerCode?}。设备：{deviceCode, text, voiceId?, providerCode?}。 后台 JWT 用户需要 ai_models.tts.view；设备通过 deviceCode 解析 tenant；设备码优先于 token。 解析 TTS provider/config/voice，返回 tts.ready，随后推送二进制 PCM 音频帧；上游完成后回 tts.done；失败回 tts.error。 实时文本转语音播放或生成 WAV Blob。 backend/config/realtime.py
backend/apps/ai_models/realtime_tts.py
tts.session.cancel 后台页面 / 设备端 -> 服务端 无 payload；可带 cancel 命令 id。 要求当前连接存在 TTS task。 取消 TTS task，返回 tts.cancelled，payload 带上一轮 sessionId；未开始则回 tts.error。 彻底停止当前合成；只停止本地播放不一定会发此命令。 backend/config/realtime.py
web/src/views/tts-realtime-playback.ts
llm.session.start 设备端 / 调试台 -> 服务端 {deviceCode, text, requestId?, traceId?}；文本字段兼容 questionText、question；设备码兼容 snake_case。 只按设备运行时身份解析，不走后台 JWT。 返回 llm.started；命中标注答案时直接回一次 llm.delta，否则流式转发 LLM delta；完成回 llm.done；失败回 llm.error。 设备智能体问答的文本阶段。 backend/config/realtime.py
device-chat/runtime-api-console.html
llm.session.cancel 设备端 / 调试台 -> 服务端 无 payload；可带 cancel 命令 id。 要求当前连接存在 LLM task。 取消 LLM task，返回 llm.cancelled，payload 带上一轮 sessionId；未开始则回 llm.error。 打断当前回答，连接仍可继续复用。 backend/config/realtime.py
web/src/api/realtime.ts
二进制帧 客户端 -> 服务端 原始 bytes，约定为 16k mono PCM。 要求当前连接已经 asr.session.start 并拿到上游 ASR session。 包装为上游 input_audio_buffer.append，audio 为 base64；没有 ASR session 时返回通用 error/media_session_not_started。 ASR 实时音频上传。TTS 音频是服务端 -> 客户端二进制，不走此方向。 backend/config/realtime.py

3. 服务端推送、回调与事件

事件 type 方向 数据结构 触发来源 主要消费者 用途 / 备注
pong 服务端 -> 客户端 {type:"pong", id, payload:{}} ping 命令。 通用 WS 客户端。 连接探活回调。
error 服务端 -> 客户端 {type:"error", id, error:{code,message}} JSON 无效、缺命令 type、未知命令、未授权、设备状态未开始、无 ASR media session 等。 所有客户端。 统一协议层错误。ASR/TTS/LLM 业务流错误另有 asr.error、tts.error、llm.error。
devices.events.subscribed 服务端 -> 后台管理端 {type,id,payload:{tenantId}} 订阅成功。 web/src/views/device-management/index.tsx 标记设备管理页实时连接可用。
devices.event 服务端 -> 后台管理端 {type:"devices.event", id:<subscribeId>, payload:<device event>} publish_device_event 将同 tenant 事件推入订阅队列。 设备管理页。 设备管理页收到 payload.type 以 device. 开头时，防抖刷新列表和统计。
device.status 服务端内部发布 -> devices.event 包装后推给订阅者 {type, tenantId, applicationId, agentApplicationId, deviceCode, status, isEnabled} device.status.start、连接关闭、同连接切换设备状态 session。 设备管理页、调试页事件日志。 设备 online/offline 的事实来源；HTTP heartbeat 只做兼容字段更新。
device.authorization 服务端内部发布 -> devices.event 包装后推给订阅者 {type, action, payload:{tenantId, applicationId, agentApplicationId, deviceCode, status, isEnabled}} 设备授权、再授权、撤销等业务服务。 设备管理页。 授权状态变化后刷新设备列表。
device.status.started 服务端 -> 设备连接 {type,id,payload:{deviceCode,status:"online"}} device.status.start 成功。 设备端、模拟器。 确认设备已被置为在线。
device.status.pong 服务端 -> 设备连接 {type,id,payload:{deviceCode}} device.status.ping 成功。 设备端、模拟器。 确认在线心跳已刷新。
asr.ready 服务端 -> ASR 客户端 {type,id,requestId,traceId} 上游 ASR WebSocket 建立并发送 session.update 成功。 ASR 管理页、应用智能体音频 hook、设备 demo、调试页。 客户端收到后开始推送 PCM 二进制分片或刷新 UI 到 listening。
asr.transcript 服务端 -> ASR 客户端 {type,id,text,originalText,replacementApplied,final} 上游 ASR transcript/delta/completed/finished 事件经过替换词规则处理。 ASR 管理页、应用智能体音频 hook、设备 demo、调试页。 text 是纠错后的文本；originalText 是原始识别；final 表示最终转写。
asr.done 服务端 -> ASR 客户端 {type,id} 上游返回 session.finished。 ASR 页面、设备端链路。 结束当前 ASR 会话，后续可进入 LLM。
asr.error / asr.cancelled 服务端 -> ASR 客户端 asr.error 带 message；asr.cancelled 带 payload.sessionId。 未授权、服务未就绪、上游异常、客户端 cancel。 ASR 页面、应用智能体音频 hook、调试页。 错误或取消本轮识别。
tts.ready 服务端 -> TTS 客户端 {type,id,sampleRate,voice} TTS 解析 voice/config 后、连接上游前发送。 tts-realtime-playback.ts、调试页。 客户端记录采样率和音色，准备播放 PCM。
二进制 PCM 音频 服务端 -> TTS 客户端 WebSocket binary frame，内容为上游 response.audio.delta base64 解码后的 PCM bytes。 上游 TTS response.audio.delta。 tts-realtime-playback.ts、调试页。 Web Audio 排队播放，或最终封装为 WAV Blob。
tts.done / tts.error / tts.cancelled 服务端 -> TTS 客户端 tts.done 带 id；tts.error 带 message；tts.cancelled 带 payload.sessionId。 上游完成、异常、客户端 cancel。 TTS 播放工具、调试页。 结束、失败或取消当前合成。
llm.started / llm.delta / llm.done 服务端 -> 设备 LLM 客户端 均带 id, requestId, traceId。started/done payload 包含设备、问题、应用和智能体信息；delta payload 为 {text}；done payload 增加 answerText。 llm.session.start 后解析设备智能体并流式请求 LLM。 运行时调试台、设备实时 manager。 设备智能体文本问答流。
llm.error / llm.cancelled 服务端 -> 设备 LLM 客户端 llm.error 带 message 或 RuntimeDeviceError payload；llm.cancelled 带 payload.sessionId。 缺设备码、缺问题、设备无可用智能体、模型不可用、LLM 异常、客户端 cancel。 运行时调试台、设备实时 manager。 失败或打断当前回答。

5. 生产客户端调用与回调

客户端位置 连接 / 发送 监听回调 数据处理 用途
web/src/api/realtime.ts 提供统一 URL、命令 id、JSON encode、ASR/TTS/cancel/unsubscribe builder。 parseRealtimeMessage(data) 只接受字符串 JSON object。 定义 RealtimeCommand、RealtimeEnvelope、RealtimeError。 后台前端统一 WebSocket API 工具层。

web/src/views/asr-management/index.tsx 打开统一 WS，设置 binaryType="arraybuffer"；onopen 发送 asr.session.start；AudioWorklet 下采样到 16k PCM 后发送 binary。 处理 asr.ready、asr.transcript、asr.done、asr.error。 实时展示原始识别、替换词纠错后文本、final 状态；停止时发送 asr.session.finish。 ASR 管理页实时测试。

web/src/views/tts-realtime-playback.ts 打开统一 WS，设置 binaryType="arraybuffer"；onopen 发送 tts.session.start，payload 含 token/tenantId/text/voiceId/providerCode。 onmessage 区分 binary 和 JSON；处理 tts.ready、tts.done、tts.cancelled、tts.error。 binary PCM 进入 WebAudio 队列播放并保存 chunks；完成时 resolve WAV Blob；abort 时发送 tts.session.cancel 并 reject AbortError。 实时 TTS 播放与导出音频。
device-chat/app.js 设备浏览器 demo 打开统一 WS；onopen 发送 asr.session.start，payload 为 {deviceCode}；ready 前音频分片进入队列。 处理 asr.ready、asr.transcript、asr.done、asr.error。 ready 后 flush 最多 80 个排队音频分片；finish 发送 asr.session.finish；完整回答仍走 HTTP voice-chat。 设备端 ASR 实时识别 demo。
web/src/api/modules/asr.ts / tts.ts 导出 buildAsrRealtimeWebSocketUrl / buildTtsRealtimeWebSocketUrl，实际都是统一 URL。 无独立回调。 兼容历史调用名称。 避免旧页面直接感知入口统一改造。

ASR 二进制方向 客户端 -> 服务端 binary 只进入 ASR 上游；没有 ASR session 时返回 media_session_not_started。 TTS/LLM 不应复用客户端二进制上行，除非服务端新增明确 media session 类型。 backend/config/realtime.py
TTS 二进制方向 TTS 音频是服务端 -> 客户端 binary PCM；JSON 事件只表达 ready/done/error/cancelled。 客户端 onmessage 必须同时处理 ArrayBuffer 和文本 JSON。 backend/apps/ai_models/realtime_tts.py

断开清理 finally 中统一 close，取消上游任务、退出上下文、移除订阅、发布 offline。 客户端直接 close 也会触发服务端清理；正常页面卸载前发送 unsubscribe/cancel 可让日志更清楚。 backend/config/realtime.py
