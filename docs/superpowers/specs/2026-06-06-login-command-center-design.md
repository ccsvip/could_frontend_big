# 登录页数字人指挥中心视觉改造设计

## 背景

当前 `/login` 已经具备可用的账号登录、账号申请、品牌展示和左侧抽象科技视觉，但整体仍偏常规浅色 SaaS 登录页。用户希望先提升登录页第一屏精致度，并指定方向为深色“数字人运维指挥中心”。

本设计只覆盖 `/login`，不改认证、权限、路由、接口和业务数据流。

## 目标

- 将登录页第一屏升级为深色指挥中心风格，提升第一眼的视觉冲击和产品质感。
- 新增一张专属 AI 主视觉图，表现数字人后台中枢、设备网络、数据流和控制面板。
- 保留现有登录与账号申请能力，避免影响后台可用性。
- 兼容桌面与移动端，确保表单可读、可点、无遮挡。

## 非目标

- 不改登录接口、账号申请接口、权限同步、默认跳转逻辑。
- 不改登录后的业务首页或后台全局布局。
- 不引入新的前端框架或重型动画库。
- 不在 AI 图片中生成中文或系统文案，避免图片文字失真。

## 视觉方向

采用“数字人运维指挥中心”方案：

- 底色：黑蓝、石墨、深青灰，形成沉浸式控制室氛围。
- 主状态色：沿用项目现有青绿色体系，但不拘泥于单一青绿色。
- 点缀色：少量冷白和琥珀色，用于状态节点、告警提示和视觉层级。
- 质感：玻璃拟态面板、细网格、数据流线、低透明边框、克制辉光。
- 气质：专业、稳定、可控，避免游戏化和过度赛博。

## 页面结构

`LoginPage` 仍作为唯一页面组件，内部布局调整为四个区域：

1. 顶部品牌区
   - 复用 `BrandMark`。
   - 在深色背景上使用浅色文字和更清晰的品牌副标题。

2. 左侧主视觉区
   - 使用新生成的横向主视觉图作为核心视觉。
   - 叠加少量代码生成的状态节点、连线或指标小面板，增强“后台中枢”感。
   - 桌面端显示为主视觉焦点，移动端降级为背景氛围或隐藏部分细节。

3. 右侧登录工作台
   - 保留用户名、密码、登录按钮、账号申请入口。
   - 改成深色半透明面板或深色背景上的高对比玻璃面板。
   - 强化输入框聚焦态、按钮层级和错误提示可读性。

4. 底部信息区
   - 保留版权信息。
   - 视觉上弱化，不抢登录表单焦点。

## AI 主视觉资产

生成一张项目内使用的登录页主视觉图，建议保存为：

`web/src/assets/login-command-center.png`

生成要求：

- 用途：登录页 hero / 背景主视觉。
- 内容：数字人运维中枢、设备网络、数据流、控制面板、抽象 AI 核心。
- 风格：高质量 3D / cinematic UI concept，深色科技控制室。
- 构图：横向宽幅，左侧或中部有视觉焦点，右侧保留可用于登录面板的暗色留白。
- 色彩：黑蓝、石墨、深青灰，青绿色状态光，少量琥珀告警光。
- 限制：无文字、无 Logo、无水印、无人脸特写、不要让画面过亮或过乱。

## 技术方案

主要修改点：

- `web/src/views/login/index.tsx`
  - 调整布局、状态文案、表单容器和主视觉层。
  - 保留 `onSubmit`、`handleApplySubmit`、表单字段和现有 API 调用。
  - 保留账号申请 Modal，统一视觉细节。

- `web/src/assets/login-command-center.png`
  - 新增 AI 生成主视觉资产。
  - 不覆盖现有 `hero.png`，避免影响其他页面引用。

如有必要，可少量补充 `web/src/styles/index.css` 的登录页专用类，但优先使用 Tailwind utility，保持项目现有风格。

## 响应式要求

- 桌面端：左右分栏，主视觉和登录面板同时可见。
- 平板端：保持双栏或压缩主视觉，表单优先。
- 移动端：登录面板优先显示，主视觉降级为背景氛围，避免内容溢出和按钮不可点。
- 不使用随视口宽度线性缩放的字体。

## 风险与约束

- AI 图片不可包含可靠文字，因此所有文案必须由 React 渲染。
- 深色页面必须保证输入框、按钮、错误提示、弹窗内容对比度足够。
- 只改 `/login`，避免把视觉改造扩散到后台业务页。
- 若 `agy` 非交互模式仍无输出，不将其作为阻塞项。
