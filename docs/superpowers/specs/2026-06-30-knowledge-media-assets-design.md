# 知识库配套素材设计

## 背景

当前智能体回答已经基于知识库召回文本内容，但知识库中的图片和视频如果依赖手工填写 URL，会给普通运营用户带来较高维护成本：素材地址变化时需要逐处修改，用户也需要理解链接、资源 ID 或标记语法。

目标是让用户以“小白可理解”的方式维护知识库配套图片和视频：用户只需要在知识库里关联现有资源管理中的图片/视频，并补充简单说明，系统在知识库回答时自动附带相关素材。

## 产品语言

面向用户统一称为“配套素材”。

推荐文案：

- 配套素材
- 关联素材
- 回答时自动附带
- 关键词
- 说明
- 启用 / 停用
- 相关度

避免在用户界面出现：

- 向量
- embedding
- rerank
- chunk
- 索引模型
- 媒体引用

## 目标

- 知识库可以关联图片和视频配套素材。
- 配套素材复用现有资源管理中的图片/视频资源，不重复上传媒体文件。
- 用户不需要在知识库正文中填写 URL、资源 ID 或特殊标记。
- 智能体回答命中知识库文本后，系统自动匹配并附带相关配套素材。
- 素材匹配失败不影响文本回答。
- 召回测试能展示知识片段和配套素材匹配结果，方便管理员调优。

## 非目标

- V1 不做以图搜图、以视频搜视频。
- V1 不分析图片像素、视频关键帧、视频音轨或字幕。
- V1 不支持 PDF、Office、音频、模型文件、滚动文本作为配套素材。
- V1 不支持全局素材自动匹配。
- V1 不做“总是附带”开关。
- V1 不做素材反馈自动学习闭环。
- V1 不做知识库级“只返回图片 / 只返回视频”配置。
- V1 不做批量导入关键词和说明。
- V1 不为普通用户展示底层索引状态。

## 核心规则

配套素材必须先绑定到某个知识库，才能参与该知识库的回答增强。

V1 只做知识库级绑定。未来可以扩展文档级绑定，但第一期不要求用户判断素材属于哪一篇文档。

同一个资源管理图片或视频可以绑定到多个知识库。每个绑定关系有独立的关键词、说明、启用状态和优先级。

删除知识库时，删除该知识库下的配套素材绑定关系，但不删除资源管理中的图片/视频本体。

删除资源管理中的图片/视频时，不阻止删除；知识库中的绑定关系保留为失效状态，并在配套素材列表中提示“素材已删除”。

## 绑定模型

建议新增知识库配套素材绑定模型，语义上对应 `Knowledge Media Asset`。

字段建议：

- 所属公司。
- 所属知识库。
- 资源管理素材。
- 素材类型快照：图片或视频。
- 关键词。
- 说明。
- 启用状态。
- 优先级。
- 失效状态或资源缺失标记。
- 创建人。
- 创建时间、更新时间。

关键词和说明存在绑定关系上，不直接写回资源本体。原因是同一个资源在不同知识库里的业务含义可能不同。

## 用户流程

知识库详情页增加“配套素材”区域，不新增一级菜单。

用户流程：

1. 用户进入知识库详情。
2. 点击“关联素材”。
3. 从当前公司资源管理的图片/视频中多选。
4. 系统为每个绑定自动生成基础描述。
5. 用户可以补充关键词、说明、启用状态和优先级。
6. 保存后，素材参与该知识库的后续回答增强。

默认描述可以由资源名称、资源分类、资源类型组合生成。用户不补充说明时，素材仍应可参与基础匹配。

## 检索流程

素材只在知识文本召回成功后参与匹配，不单独触发回答。

回答流程：

1. 用户提问。
2. 在智能体绑定的知识库中召回知识文本。
3. 根据命中的知识片段确定素材匹配范围。
4. 在命中知识片段所属知识库的配套素材中匹配素材。
5. 使用“用户问题 + 命中知识片段”作为素材匹配依据，其中知识片段权重更高。
6. 过滤低于最低相关度阈值的素材。
7. 把候选素材的名称、类型、说明提供给 LLM。
8. LLM 只生成文本回答。
9. 后端附带真实图片/视频 `Agent Reply Block`。

如果一个智能体绑定了多个知识库，只在已命中的知识片段所属知识库内匹配素材。多个知识库都命中时，素材名额按知识片段相关性分配，不按知识库平均分配。

## 返回规则

默认每次回答最多返回 3 个素材：

- 图片最多 2 个。
- 视频最多 1 个。
- 素材必须超过最低相关度阈值。
- 主排序按相关性。
- 图片默认优先；视频只有在相关度明显高于图片时才前置。
- 优先级只在相关性接近时作为辅助排序。

LLM 可以知道候选素材的名称、类型和说明，用于自然表达，但不能决定真实资源 ID 或 URL。真实图片/视频 block 由后端根据检索结果和权限校验附带。

即使 LLM 文本没有主动提到图片或视频，后端也可以附带已经匹配到的素材。

## 日志与快照

自动附带的素材需要保存到聊天消息日志中，方便复盘。

保存建议：

- 资源 ID。
- 资源名称。
- 资源类型。
- 知识库 ID。
- 知识库名称。
- 相关度。
- 命中来源或原因摘要。
- 创建时间。

不要保存真实 URL 或长期预签名地址。渲染时应根据资源 ID 重新获取当前可访问 URL。

V1 不做素材反馈自动学习闭环。管理员可以通过日志观察错误匹配，再回到知识库配套素材中调整关键词、说明、启用状态或优先级。

## 召回测试

知识库召回测试需要同时展示配套素材匹配结果。

测试结果建议展示：

- 命中的知识片段。
- 自动匹配到的配套素材。
- 素材相关度。
- 因低于阈值未附带的素材数量。

面向管理员使用“相关度”文案，不展示技术分数名称。

## 权限与租户隔离

配套素材必须严格跟随租户隔离。

规则：

- 只能绑定当前公司自己的资源。
- 只能从当前公司知识库里召回。
- 超管公司视图必须按当前 tenant scope 操作。
- 运行时返回素材 URL 前必须再次校验素材、知识库、智能体属于同一公司作用域。
- 失效或已删除的资源不能返回给前端或设备端。

## API 与前端同步点

后端建议在知识库资源下提供配套素材子集合，保持 REST 风格。

示例路径：

- `GET /api/v1/knowledge-bases/{id}/media-assets/`
- `POST /api/v1/knowledge-bases/{id}/media-assets/`
- `PATCH /api/v1/knowledge-bases/{id}/media-assets/{assetId}/`
- `DELETE /api/v1/knowledge-bases/{id}/media-assets/{assetId}/`

前端知识库详情页增加“配套素材”区域，并在召回测试中展示素材匹配结果。

智能体聊天链路继续返回现有 `Agent Reply Block` 结构，避免前端和设备端接入第二套媒体响应格式。

## 风险

- 如果素材关键词和说明过少，自动匹配会不稳定；需要默认描述和召回测试帮助用户调优。
- 如果不记录日志快照，错误匹配很难排查。
- 如果把素材匹配范围扩大到智能体绑定的全部知识库，容易跨主题返回错误图片或视频。
- 如果让 LLM 直接输出资源 ID 或 URL，容易产生幻觉、越权或过期链接。
