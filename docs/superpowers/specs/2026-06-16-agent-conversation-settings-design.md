# 智能体对话设置增强设计

## 背景

当前智能体工作室已经具备创建、编排、知识库、调试预览、日志与监测能力，但详情页仍偏“模型配置”，缺少面向真实对话体验的设置。用户反馈主要集中在：

- 创建的智能体无法删除。
- 进入编排页后功能太少。
- 缺少 ASR 语音输入。
- 缺少 TTS 播报、暂停和继续。
- 不支持开场白和建议问题。

经代码核对，删除功能前后端已存在，但公司租户默认权限没有发放 `agent_applications.delete`，导致普通公司用户和公司管理员无法删除。现有 `ChatConversation.application` 是 `SET_NULL`，删除智能体后会保留历史会话，这也不符合用户确认的“智能体是工具箱，工具箱删除后内部历史也删除”的产品语义。

## 目标

- 修复智能体删除权限，让公司用户在授权范围内可以删除自己公司下的智能体。
- 删除智能体时删除该智能体关联的会话和消息，并给出明确不可恢复警告。
- 详情页新增独立 `对话设置` 分区。
- 支持配置开场白。
- 支持配置最多 10 条建议问题。
- 支持智能体调试中的 ASR 语音输入。
- 支持开场白和助手回复的 TTS 播放、暂停、继续和停止。
- 保存对话设置后，右侧调试预览即时刷新。

## 非目标

- 不删除知识库文档、LLM 模型、TTS 音色、ASR/TTS 平台或公司配置。
- 不做用户输入播报。
- 不做建议问题按钮文本播报。
- 不做实时语音打断、边说边发、LLM 生成中断联动 TTS/ASR。
- 不支持每个智能体单独选择 TTS 音色；V1 使用公司默认 TTS 音色。
- 不把开场白保存为 `ChatMessage`。
- 不自动生成建议问题。

## 产品语义

### 智能体是工具箱

智能体本身是一个可删除的工具箱。删除智能体时：

- 删除 `AgentApplication`。
- 删除该智能体下的 `ChatConversation`。
- 删除这些会话下的 `ChatMessage`。

以下资源不删除：

- `KnowledgeDocument`。
- `LLMModel` / `LLMProvider`。
- `TTSVoice` / `TTSProvider`。
- `ASRConfig` / `TTSProvider` 平台配置。
- 公司默认 TTS 音色。

删除弹窗必须明确提示：删除会移除智能体配置、对话设置、关联会话和消息，且不可恢复；绑定的知识库、模型、音色不会删除。

### 对话设置属于智能体

开场白、建议问题、ASR 输入开关、TTS 自动播报开关都属于 `AgentApplication` 的运行时对话设置，不属于设备，也不是用户个人偏好。后台调试预览、后续设备运行和外部调用应使用同一套设置。

## 页面结构

智能体详情页左侧内部菜单调整为四项：

- `编排`：模型、提示词、知识库、生成参数。
- `对话设置`：开场白、建议问题、ASR 输入、TTS 回复播报。
- `日志与标注`：会话和消息记录。
- `监测`：统计数据。

`对话设置` 页面参考用户提供的界面：

- 左侧为设置表单。
- 右侧为调试预览。
- 设置保存后预览区即时刷新。
- 如果当前有未完成的流式回复，保存按钮禁用，或提示先停止生成。

## 对话设置

### 开场白

新智能体默认启用开场白。

默认模板：

```text
你好，我是{智能体名称}，很高兴见到你，有什么我可以帮你的吗？
```

规则：

- 开场白只作为配置展示和播报。
- 不保存为 `ChatMessage`。
- 不计入消息统计。
- 修改后预览区立刻显示新内容。
- 支持手动播放。
- 如果开启自动播报，进入新调试会话时可自动播放开场白。

### 建议问题

建议问题默认为空。

规则：

- 最多 10 条。
- 支持新增、编辑、删除、排序。
- 点击建议问题后直接发送，不只是填入输入框。
- 建议问题文本不播报。
- 点击发送后创建真实用户消息，走普通聊天流程。
- 如果配置未保存，点击建议问题前应提示保存，或先保存后发送；推荐先提示保存，避免隐藏写入。

### ASR 输入

ASR 输入默认关闭。

规则：

- 如果 ASR 服务未配置或不可用，开关置灰，并提示去 ASR 设置。
- 开启后，调试输入区显示麦克风按钮。
- V1 采用录音转文字再发送：
  - 点击麦克风开始录音。
  - 再点停止。
  - 后端返回识别文本。
  - 文本填入输入框。
  - 用户确认后再发送。
- ASR 失败只提示错误，不创建消息。
- 不做实时打断、不做边说边发。

实现上优先复用现有 `/ws/asr/test/` WebSocket 链路和 `asr-management` 中的前端录音、PCM 下采样、AudioWorklet 逻辑。

### TTS 播报

TTS 自动播报默认关闭。

规则：

- 手动可播放开场白。
- 手动可播放每条助手回复。
- 只播放开场白和助手回复。
- 不播放用户输入。
- 不播放建议问题按钮文本。
- 播放控件支持播放、暂停、继续、停止。
- 开启自动播报后：
  - 开场白可自动播放。
  - 新助手回复完成后自动播放。
  - 不重播历史消息。
- 使用公司默认 TTS 音色。
- 如果公司未配置默认 TTS 音色或 TTS 服务不可用，播放按钮置灰并提示去 TTS 设置。

实现上优先复用现有公司 TTS 合成接口能力，不在前端直接接触供应商密钥。

## 后端设计

### 数据模型

在 `AgentApplication` 上增加对话设置字段，建议字段：

- `opening_message_enabled: bool`，默认 `true`。
- `opening_message: text`，默认使用名称模板。
- `suggested_questions: json/list[str]`，默认空数组，最多 10 条。
- `voice_input_enabled: bool`，默认 `false`。
- `reply_playback_enabled: bool`，默认 `false`。

删除语义：

- 将 `ChatConversation.application` 从 `on_delete=SET_NULL` 调整为 `on_delete=CASCADE`，或在删除 `AgentApplication` 前显式删除关联会话。
- 推荐用数据库级 `CASCADE`，语义更贴近“会话属于智能体”，也避免绕过 view 删除时留下孤儿会话。

### 序列化与校验

`AgentApplicationSerializer` 增加字段：

- `openingMessageEnabled`
- `openingMessage`
- `suggestedQuestions`
- `voiceInputEnabled`
- `replyPlaybackEnabled`

校验规则：

- `suggestedQuestions` 最多 10 条。
- 每条建议问题 trim 后不能为空。
- 建议问题建议限制单条长度，例如 120 字以内。
- `openingMessage` 建议限制长度，例如 200 字以内。

创建规则：

- 创建应用时如果未传 `openingMessage`，按应用名称生成默认开场白。
- 建议问题默认空。
- ASR 和自动 TTS 默认关闭。

### 删除权限

修复租户默认权限种子：

- 默认公司权限应包含 `agent_applications.delete`。
- 公司管理员和普通员工在公司授权范围内权限一致，除员工管理外不再分裂。
- 相关测试需要覆盖默认租户拥有删除权限。

### ASR 状态

前端需要知道 ASR 是否可用：

- 复用 `GET /api/v1/ai-models/asr/status/`。
- `voiceInputEnabled` 只是智能体开关；实际能否使用还取决于 ASR 服务状态。

### TTS 播放

前端需要生成可播放音频：

- 复用公司 TTS 测试/合成接口，输入为要播放的文本。
- 返回浏览器可播放的 WAV 音频更适合后台预览。
- 权限使用现有 `ai_models.tts.view`。

如果现有接口语义过于“测试”，实现时可以增加更准确的管理端播放接口，但仍使用公司默认 TTS 配置和默认音色。

## 前端设计

### 智能体列表

删除入口保持在卡片上，但需要：

- 只在有 `agent_applications.delete` 权限时显示。
- 删除确认文案明确说明会删除关联会话和消息。
- 删除成功后刷新列表。

### 详情页

新增 `对话设置` tab。

表单区：

- 开场白开关。
- 开场白文本框。
- 建议问题列表，最多 10 条。
- 新增建议问题输入。
- 删除、排序建议问题。
- ASR 输入开关和服务状态提示。
- 自动 TTS 播报开关和服务状态提示。
- 保存按钮。

预览区：

- 没有真实消息时展示开场白。
- 展示建议问题按钮。
- 输入区在 ASR 开启且服务可用时显示麦克风按钮。
- 开场白旁提供播放/暂停/继续/停止控件。
- 每条助手回复旁提供播放/暂停/继续/停止控件。
- 点击建议问题后直接发送。

### 状态同步

- 保存对话设置后，当前页面状态和预览立即更新。
- 若当前正在流式生成，保存按钮禁用或提示先停止。
- 切换智能体时清空当前播放状态、录音状态和调试会话状态。
- 播放新文本前停止当前正在播放的音频，避免多段声音叠加。

## 测试与验证

后端测试：

- 新建智能体默认启用开场白，默认开场白包含智能体名称。
- 新建智能体建议问题为空。
- 建议问题最多 10 条，超过返回 400。
- 删除智能体会删除关联 `ChatConversation` 和 `ChatMessage`。
- 删除智能体不会删除知识库文档、模型、音色或 TTS/ASR 配置。
- 默认租户权限包含 `agent_applications.delete`。

前端验证：

- 有删除权限时列表卡片显示删除入口。
- 删除弹窗文案明确提示会删除会话和消息。
- `对话设置` 能保存开场白、建议问题、ASR 开关、TTS 开关。
- 建议问题超过 10 条时不能继续新增。
- 点击建议问题直接发送消息。
- ASR 未配置时麦克风不可用。
- TTS 未配置时播放按钮不可用。
- TTS 播放支持暂停、继续、停止。
- 设置保存后预览即时刷新。

验证命令：

```bash
docker compose exec backend python manage.py test apps.ai_models.tests.test_agent_application_api
docker compose exec backend python manage.py test apps.ai_models.tests.test_tts_api
docker compose exec web npm run build
```

## 风险

- `application=CASCADE` 会改变删除语义，必须通过强警告降低误删风险。
- ASR 前端录音逻辑较复杂，应尽量复用现有 `asr-management` 的稳定代码，避免复制后出现分叉。
- TTS 浏览器播放需要正确释放 object URL，避免长时间调试产生内存泄漏。
- 自动播报可能和用户手动播放冲突，播放控制需要保证同一时间只有一个音频实例。
- 建议问题点击直接发送前必须处理未保存配置，否则用户可能以为新建议问题已经生效。
