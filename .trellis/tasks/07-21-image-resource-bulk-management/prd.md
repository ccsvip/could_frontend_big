# 图片资源批量管理与删除

## Goal

在 `/resources/images` 图片资源页面提供当前列表范围内的多选与批量删除能力，减少逐张删除的重复操作，同时保持现有权限、公司隔离和资源引用保护规则。

## Background

- 图片和视频共用 `Resource` 模型，但本次仅修改图片资源页面与图片 API。
- 当前仅支持单项删除；图片页面已有批量上传功能。
- 单项删除会阻止删除仍被智能体标注回复引用的资源。
- 图片列表按当前登录公司的 tenant scope 返回，普通公司账号不得操作其他公司资源。

## Requirements

- 图片卡片支持多选，并提供当前页全选、取消选择和已选数量反馈。
- 仅拥有 `resources.images.delete` 权限的用户可进入批量管理和执行批量删除。
- 批量删除前必须显示确认信息，明确本次选择数量及不可恢复性。
- 后端提供图片资源集合上的 REST 批量删除能力，请求只接受稳定资源 ID 列表。
- 后端必须通过现有 tenant scope 解析待删除资源，越权或不存在的 ID 不得被删除。
- 批量采用部分成功策略：可删除项立即删除，不可删除项保留并逐项返回失败原因。
- 每个资源继续执行现有引用保护、存储对象清理和业务缓存失效逻辑。
- 删除完成后前端清空已删除项选择，刷新当前列表；若当前页被删空且不是第一页，回退到上一页。
- 请求和响应字段使用 camelCase，错误必须可定位到具体资源。

## Acceptance Criteria

- [ ] 有删除权限时，用户可进入批量管理并选择单张、多张或当前页全部图片。
- [ ] 无删除权限时，不显示批量删除入口，直接调用接口返回 403。
- [ ] 确认批量删除后，可删除项不再出现在列表中，分页总数同步更新。
- [ ] 其他公司的资源 ID、视频资源 ID和不存在的 ID不会被删除。
- [ ] 被智能体标注回复引用的图片仍受到保护，响应能指出失败资源及原因。
- [ ] 批量删除触发资源业务缓存失效，并沿用本地文件/对象存储删除语义。
- [ ] 相关 Django API 测试通过，前端 `npm run build` 通过。

## Out Of Scope

- 跨页全选全部搜索结果。
- 批量修改分类、用途或描述。
- 视频资源批量管理。
- 恢复已删除资源或回收站。
