# 智能体回答知识引用展示

## Goal

在智能体管理网页中，为每一条使用知识库召回结果生成的助手回答展示可追溯的“知识引用”。该能力同时覆盖网页调试对话和设备运行时对话，帮助管理员确认本轮回答检索了哪些知识库、文档与切片，并在刷新或稍后查看历史记录时保持一致。

## Background

- 网页调试入口位于 `/ai-models/applications/{applicationId}`，现有聊天通过 SSE 返回回答正文、内容块、标题、摘要和后续建议问题，但未返回知识引用。
- 设备运行时可能通过 HTTP 或统一 WebSocket 发起对话；两条链路最终写入 `DeviceChatLog`，管理网页通过设备会话记录查看历史消息。
- `retrieve_knowledge_chunks()` 已返回结构化召回结果，包括知识库、文档、切片正文、相关度和顺序；当前调用方主要消费拼接后的模型上下文，引用元数据未随回答持久化。
- `ChatMessage` 与 `DeviceChatLog` 当前分别持久化回答内容块，但没有独立的知识引用快照字段。
- Dify 将本轮召回资源作为助手消息的独立结构化数据持久化；即时聊天和历史接口使用同一引用语义，前端按文档聚合展示。研究记录见 `.scratch/dify-knowledge-citations-research.md`。

## Product Decisions

- **展示范围**：仅管理网页展示，不要求安卓或其他设备端展示。
- **设备协议兼容**：不向设备 HTTP 响应或统一 WebSocket 事件新增知识引用字段，不改变现有 `Device Chat Contract`。
- **数据归属**：引用属于产生回答的助手消息或设备交互日志，必须持久化，不能只存在于 SSE 临时状态。
- **展示语义**：文案使用“知识引用”或“检索来源”，不宣称每条来源都被模型逐字采用。
- **历史稳定性**：保存召回时的名称、正文和分数快照；知识库文档后续修改或删除不改变旧对话中已经记录的引用内容。
- **内容隔离**：引用不混入 `contentBlocks` / `answerBlocks`，避免影响回答渲染、TTS 和设备播放。
- **引用范围**：只保存实际拼入模型上下文的切片；被 `max_chars` 截断、未送入模型的召回候选不展示。

## Requirements

### R1 — 统一引用契约

- 定义一套管理端 camelCase 引用 DTO，至少包含：顺序、知识库 ID/名称、文档 ID/名称、切片标识或序号、切片正文和相关度。
- 网页调试即时事件、网页调试历史消息和设备会话历史消息返回相同字段结构。
- 所有 ID 与数据必须来自当前租户已授权且实际参与本轮召回的本地映射，不暴露百炼 Workspace、Index、File 等远端标识。

### R2 — 网页调试对话

- 平台 LLM 回答完成时，将本轮引用保存到对应 `ChatMessage`。
- SSE 在回答完成阶段返回引用数组，前端可在不额外发起召回的情况下立即展示。
- 重新加载会话时，历史消息 API 返回相同引用数组。
- 标注命中、第三方机器人或未发生知识检索的回答不显示引用。
- 重生成最后一条回答时，新回答只关联本次重新召回的引用，旧回答及其引用一并按现有重生成语义移除。

### R3 — 设备运行时对话

- 设备通过 HTTP 或统一 WebSocket 发起平台 LLM 对话时，将本轮引用保存到对应 `DeviceChatLog`。
- 管理网页读取设备会话历史时，在助手消息上返回引用数组。
- 设备请求与实时响应契约保持原样，不向设备返回知识引用。
- 标注命中、第三方机器人、控制指令直达或未发生知识检索时保存空引用。

### R4 — 管理网页展示

- 仅在助手回答存在引用时，于回答正文下方显示紧凑的引用入口。
- 首层按文档聚合，显示引用文档数量及文档名称；同一文档命中多个切片不重复占用首层位置。
- 点击引用入口后可查看每个命中切片的正文预览、相关度和所属知识库/文档。
- 网页调试消息与设备会话消息复用同一引用展示组件。
- 组件遵循现有 `brand-*`、`text-fluid-*`、响应式和 Tabler Icons 规范。

### R5 — 可靠性与隔离

- 引用采集失败不能导致回答失败；记录可诊断日志，并以空引用降级。
- 引用数据随所属聊天消息或设备日志一起受 tenant scope 保护。
- 不通过客户端参数查询或拼接其他租户的知识库、文档或切片。
- 空引用、缺失名称、旧历史记录无引用字段等情况必须兼容。

## Acceptance Criteria

- [ ] **AC1** 网页调试问题命中知识库后，回答完成时显示引用；刷新会话后引用仍然存在且内容一致。
- [ ] **AC2** 同一文档命中多个切片时，首层只显示一个文档入口，展开后能看到全部已保存切片。
- [ ] **AC3** 设备通过 HTTP 发起并命中知识库后，设备响应保持原契约，管理网页的设备会话记录可查看引用。
- [ ] **AC4** 设备通过统一 WebSocket 发起并命中知识库后，WebSocket 事件保持原契约，管理网页的设备会话记录可查看引用。
- [ ] **AC5** 未检索、标注回复、第三方机器人和控制指令直达回答不显示错误的知识引用。
- [ ] **AC6** 重生成网页回答后只显示新一轮召回引用，不残留被删除回答的引用。
- [ ] **AC7** 旧消息和旧设备日志没有引用数据时，历史页面正常渲染。
- [ ] **AC8** 普通公司账号只能读取本公司对话引用，平台管理员浏览公司数据时继续遵循现有 tenant scope。
- [ ] **AC9** 引用采集或序列化异常不阻断 LLM 回答、SSE 完成、设备 HTTP 响应或 WebSocket 响应。

## Out of Scope

- 不实现答案正文中的 `[1]` 行内精确标注或逐句来源归因。
- 不要求设备端、安卓端展示引用。
- 不向第三方机器人要求或推断其内部知识库引用。
- 不新增知识库召回、重排或相关度算法。
- 不允许从引用入口编辑或删除知识库切片。
