页面用于后端给安卓同学对接口：每个模块都包含接口用途、请求示例、可执行测试和响应展示。

重要约定：安卓设备请求不携带 Authorization/JWT；后端通过 X-Device-Code 找设备，再解析公司、资源应用和绑定智能体上下文。图片、模型、视频来自资源应用；ASR、LLM、TTS/语音问答按设备所属公司和绑定智能体配置执行。本页会把运行时配置拆开展示给安卓对接。

Step 1

设备激活 / 上报

POST/device-auth/activate/

安卓用途

设备启动后上报设备信息。后端会根据设备码返回绑定状态；未登记设备会进入待授权流程。

固定 Header

X-Device-Code: DEVICE_001
Content-Type: application/json
等待请求

Step 2

获取运行时配置

GET/device-runtime/config/?deviceCode=DEVICE_001

安卓用途

启动或切换设备状态后拉取应用与资源配置。语音问答就绪度以“绑定公司 + 可用智能体应用”为准；资源应用只影响图片、模型、视频、音色、滚动文本等 resources 展示。

ASR

ASR 状态与实时协议

GET/ai-models/asr/device-status/WS/ws/realtime/

实时 ASR Start 命令

{
  "type": "asr.session.start",
  "id": "asr-session-xxx",
  "payload": {
    "deviceCode": "DEVICE_001"
  }
}

安卓音频要求

  • 连接统一入口 /ws/realtime/
  • 收到 asr.ready 后发送 16k PCM 二进制分片
  • 监听 asr.transcript / asr.done / asr.error

LLM

LLM 文本问答

POST/device/voice-chat

说明

管理后台的 LLM 能力可独立配置和测试；安卓设备运行时这里使用 /device/voice-chat 作为设备问答入口。传 text 时后端会按设备绑定智能体执行 LLM，并尝试合成 TTS，适合安卓调试完整问答链路。

TTS

TTS 单独合成

安卓注意

该接口返回 raw PCM，响应头包含采样率、声道和音色。浏览器会尝试封装 WAV 播放；安卓端可直接按响应头播放 PCM。
