# 公司 ASR 语气词词表

Status: ready-for-agent

## Problem Statement

当前 ASR 语气词过滤以固定字符集合判断文本。由语气字符组成的完整短回复会被当成无意义内容而丢弃，例如“嗯嗯”和“哦呀”。同时，过滤开关属于平台级 ASR 配置，不能满足不同公司拥有不同过滤词的需求，也无法让公司明确控制哪些单字应被抑制。

## Solution

在公司 `/ai-models/asr` 页面提供公司 ASR 语气词词表输入框。管理员以换行分隔填写需要过滤的单个汉字；空行被忽略，重复项自动去重。实时 ASR 仅在完整识别结果经空白和标点规范化后，精确等于词表中的一项时抑制该结果。

词表是公司级数据，默认为空。移除平台级“过滤语气词”总开关；公司词表非空时，才会按该公司的词表过滤。现有公司不自动写入历史默认词。

## User Stories

1. 作为公司管理员，我希望在 ASR 页面按行维护本公司的语气词词表，以便只过滤本公司明确不需要的单字识别结果。
2. 作为公司管理员，我希望词表默认为空，以便升级后不会意外丢弃语音内容。
3. 作为公司管理员，我希望空行和重复项被自动忽略，以便词表保持简洁且无需手动清理。
4. 作为公司管理员，我希望系统拒绝多字或非汉字条目，以便不能把有语义的短句误设为过滤项。
5. 作为设备使用者，我希望“嗯。”在公司配置“嗯”后不被发送，以便无意的单字语气词不触发后续处理。
6. 作为设备使用者，我希望“嗯嗯”被保留，以便重复表达不被误判为单个语气词。
7. 作为设备使用者，我希望“哦呀”被保留，以便组合短回复仍能进入后续会话处理。
8. 作为设备使用者，我希望“嗯，好的”被保留，以便包含实际语义的句子不被抑制。
9. 作为设备使用者，我希望被抑制的最终 ASR 结果仍能正常结束实时识别，以便三合一客户端不会卡在录音或等待状态。
10. 作为公司管理员，我希望其他公司的词表不会影响本公司的 ASR 结果，以便公司数据和运行时行为保持隔离。
11. 作为平台管理员，我希望在现有公司作用域机制下查看或维护目标公司的词表，以便不绕过既有租户权限模型。
12. 作为前端用户，我希望不再看到平台级“过滤语气词”开关，以便页面只呈现实际生效的公司级配置。
13. 作为运维人员，我希望运行时对没有词表的公司不做语气词抑制，以便行为清晰且可预测。

## Implementation Decisions

- 引入 Company ASR Filler Word Set：公司范围、换行分隔、单个汉字的过滤词集合。
- 词表通过现有公司作用域和权限模型读写；普通公司账号不得通过参数读取或修改其他公司数据，平台管理员沿用既有 tenant scope。
- 新增或扩展 REST 资源时使用稳定的名词资源路径，并保持前端 camelCase 与后端 serializer 映射惯例。
- 保存时忽略空行、移除首尾空白、去重；每个非空条目必须恰好为一个汉字。无效条目返回可定位的字段校验错误。
- 词表默认空，不将旧固定词或历史开关状态写入任何公司数据。
- 删除平台 ASR 配置中的 `filterFillerWords` 和后端等价字段、序列化字段、运行时配置字段与页面总开关，不保留双重开关语义。
- 语气词匹配只比较完整识别结果：先执行现有 ASR 替换规则，再忽略空白和标点，随后与当前公司词表中的单项做精确匹配。
- 不得以字符集合、包含匹配、重复字符折叠或正则片段匹配替代完整精确匹配；“嗯嗯”“哦呀”“嗯，好的”不得因“嗯”“哦”“呀”被配置而抑制。
- 对实时 ASR 的最终事件，命中词表时不下发 `asr.transcript`，但必须按与普通最终事件等价的收尾协议结束上游会话并最终下发完成事件，防止三合一客户端持续等待。
- 对非最终增量事件，命中词表时可不下发该增量，但不得影响后续最终事件的正常处理和会话收尾。
- 页面在 `/ai-models/asr` 展示多行输入区域、单字约束说明与示例；页面保持现有设计 token、流体字体和移动端布局规范。

## Testing Decisions

- 测试关注可观察契约：保存后的公司词表、ASR 是否下发转写、最终事件是否完成；不断言内部集合或辅助函数的实现细节。
- 在现有 ASR 配置 API 测试缝隙中覆盖：公司保存和读取自己的词表、跨公司隔离、空行与重复项处理、单字汉字校验、默认空词表，以及移除平台总开关后的 API 契约。
- 在现有实时 ASR 测试缝隙中覆盖：配置“嗯”后 `嗯。` 被抑制；`嗯嗯`、`哦呀`、`嗯，好的` 被下发；被抑制的最终事件仍发出完成信号并关闭接收音频状态。
- 复用现有 ASR 替换规则测试模式，验证替换后的完整文本才是匹配对象，避免替换与过滤顺序回归。
- 使用后端 Docker Compose 环境运行相关 Django 测试；前端变更后在 `web/` 运行 `npm run build`。

## Out of Scope

- 不支持词组、句子、正则表达式、包含匹配或按字符集合过滤。
- 不支持英文语气词、拼音语气词或多字中文条目。
- 不迁移或猜测任何公司应继承的历史默认词。
- 不修改 ASR 供应商、音频格式、VAD 参数、替换规则的业务语义或统一 WebSocket 入口。
- 不新增业务专用 WebSocket URL。

## Further Notes

- Company ASR Filler Word Set 的领域定义已写入 `CONTEXT.md`。
- 三合一场景的关键验收点是：被抑制的最终识别结果不能让客户端的录音或等待状态悬挂。
