# Product

## Register

product

## Users
开发与运维人员、多租户管理员。他们需要在日常工作中频繁管理 AI 模型供应商、配置全局 API 密钥凭证，以及为各个租户/公司进行可用模型授权与测速冷却策略设定。界面需要保证高效率、直观和低操作负担。

## Product Purpose
数字人后台管理平台（solin）的大语言模型（LLM）配置中心，支持平台级供应商账号接入、多租户模型精细授权以及 API 可用性自动化连通测试，确保数字人智能体拥有稳定可靠的后端大模型服务支撑。

## Brand Personality
科技、专业、高效。设计应当冷静克制，利用高对比度、清晰的布局和逻辑化的信息呈现，展现出平台系统的安全感与技术专业性。

## Anti-references
- 避免过度圆角化（Card 等组件圆角不超过 12px-14px）。
- 避免使用装饰性的渐变文字或非功能性的毛玻璃效果。
- 避免引入冲突的色系（例如非品牌色的紫色、荧光色等），坚持以 Teal 青绿色为主色调。
- 避免 SaaS 模板式的夸张卡片阴影与超大卡片间距，确保紧凑高效。

## Design Principles
1. **设计服务功能**：界面的每一个元素都必须有助于用户完成操作，禁止为了美观加入干扰注意力的装饰物。
2. **色彩传达状态**：以 Teal（青绿色）作为品牌基准色，只在状态表达（如启用、停用、成功、失败）时使用清晰对应的语义色彩（绿、灰、绿、红）。
3. **等宽对齐与易读**：密钥、API 地址、模型名等技术标识统一采用等宽字体 (`font-mono`) 并支持一键选中，减少识别与输入误差。

## Accessibility & Inclusion
- 文本对比度必须符合 WCAG AA 标准，文字与背景色对比度不低于 4.5:1。
- 交互组件应支持清晰的焦点状态，保证无障碍阅读。
