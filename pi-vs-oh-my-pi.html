Pi vs Oh My Pi

两大 AI 编码助手的完整对比 — 哲学、架构、功能、生态

Pi

极简 · 可塑 · 敏捷

Mario Zechner 创建的轻量终端编码助手。天生极简：仅 4 个内置工具（read / write / edit / bash）、不到 1000 token 的 system prompt。不预置 sub-agent、plan mode、MCP、权限弹窗——所有能力通过 Extension 按需加载。

pi.dev · GitHub
VS

Oh My Pi (OMP)

电池全包 · IDE 有线 · 生产就绪

can1357 从 Pi fork 并重写的"编码优先"分支。将原本的轻量脚手架膨胀为完整的生产工具：~55k 行 Rust 引擎、32 个内置工具、40+ 模型提供商、Hashline 编辑、LSP 集成、DAP 调试器、Sub-agent 编排。

omp.sh · GitHub
ⓘ 关系说明：Oh My Pi 是 Pi 的一个深度 fork，并非独立项目。两者代码同源（TypeScript 核心 + Bun 运行时），但 OMP 重写了 Rust 底层引擎并大幅膨胀了工具面。当前最新：Pi v0.74+ / OMP v17.0.7（2026-07-21）。

🔬 三、关键差异化功能
功能PiOh My Pi
Hashline 编辑 ❌ 无 ✅ 核心特性。每行代码绑定内容 hash，编辑时引用 hash 而非行号，避免行偏移/空白变更导致的编辑失败。提升首次编辑成功率 6–68 个百分点。
LSP 集成 ❌ 无原生 LSP ✅ 深度集成。14 种 LSP 操作：diagnostics / references / rename（含 workspace/willRenameFiles，自动更新 re-export / barrel / alias）、code_actions / definition / hover / symbols ……
DAP 调试器 ❌ 无 ✅ 原生 DAP 调试。28 种 DAP 操作，支持 lldb-dap（C/C++）、dlv（Go）、debugpy（Python）等真实调试器 attach — 断点、单步、栈帧、变量检查——不仅仅是打印日志。
AST 编辑 ❌ 无 ✅ ast_edit（基于 tree-sitter 的结构感知改写）+ ast_grep 搜索
代码审查图谱 ❌ 无 ✅ Code Review Graph（community detection、impact radius、bridge nodes、test coverage gap）
浏览器自动化 ❌ 无 ✅ 内置 Puppeteer/Playwright 浏览器控制
Sub-agent 编排 ❌ 默认无（可通过 Extension 加） ✅ 内置 task/sub-agent 系统，isolated worktree，并行 fan-out
Web 搜索 ❌ 无 ✅ 内置 web_search 工具
持久记忆 可通过 Extension 实现 ✅ mnemopi 引擎
进程入口模式 （交互 / 单发 / RPC / SDK）4 种 interactive / one-shot / RPC / ACP，支持 Node/TypeScript 嵌入

💡 五、如何选择

选 Pi 如果……

  • 你想要一个极简的起点，自己决定能力边界
  • 你对 context 窗口非常敏感，想要最小的 system prompt
  • 你喜欢自己组装工具链，不喜欢预置的"全家桶"
  • 你主要在终端工作，不需要 LSP / DAP / Hashline
  • 你想要通过编写 TypeScript Extension 来深度定制
  • 你对"开箱即用"没有要求，愿意投入配置时间换取灵活性

选 Oh My Pi 如果……

  • 你想要开箱即用的生产级工具，不愿折腾配置
  • 你需要 IDE 级特性：LSP rename、代码审查、实时调试
  • 你想用 Hashline 避免文件编辑因行号变化而失败
  • 你经常需要多模型路由（不同任务用不同模型）
  • 你需要 sub-agent 并行执行和隔离 worktree
  • 你想在终端里 debug 真实进程、检查栈帧和变量
  • 你喜欢活跃的社区和快速迭代
📅 发展时间线
~2024 中
Mario Zechner 创建 Pi
作为个人项目，以最小化 agent harness 为理念，只有 4 个工具和不到千 token 的 system prompt。
~2025 初
can1357 fork → Oh My Pi
从 Pi fork，目标是"编码优先的表面"——增加大量内置工具、Rust 引擎、多提供商支持。
2025–2026
两者并行演化
Pi 保持极简哲学，迭代稳定；OMP 快速膨胀，引入 Hashline、LSP、DAP、sub-agent、code-review-graph 等重量级特性。
2026-07-21
OMP v17.0.7 发布
版本号 17 表明 OMP 的迭代速度远超原版 Pi（后者处于 v0.74.x）。
💬 一句话总结：Pi 是"一把你可以自己磨的刀"，Oh My Pi 是"一把出厂开好刃、带刀鞘和磨刀石的瑞士军刀"。两者同源，但面向完全不同的开发哲学——Pi 把控制权给你，OMP 把生产力给你。
